# AI Game Blind Arena Legacy Service Spec

이 문서는 현재 레거시 서비스의 실제 구현 기준 설계 문서다.
목적은 신규 `AI_Game` 루트로 React + TypeScript 기반 마이그레이션을 진행할 때
기능, 화면, 정책, 데이터, API, 예외 동작이 누락되지 않도록 기준선을 고정하는 것이다.

기준 범위:
- 프런트엔드: `backend/static`
- 백엔드: `backend/main.py`, `backend/utils.py`, `backend/models.py`, `backend/services.py`, `backend/database.py`
- DB 스키마: `backend/schema.sql`

## 1. 서비스 개요

서비스명:
- `AI Game Blind Arena`

핵심 목적:
- 동일한 프롬프트로 생성된 여러 AI 모델의 게임 결과물을 블라인드 상태로 플레이하고 평가하는 서비스

핵심 사용자 가치:
- 모델별 게임 결과물 비교
- 블라인드 공정 평가
- 사용자별 마이페이지 통계
- 배지 획득 및 프로필 꾸미기
- 커뮤니티형 코멘트/댓글 반응

현재 계정 구조:
- 정식 계정 시스템 없음
- 닉네임 기반 진입
- 동일 닉네임 소유권 검증 없음
- 관리자만 별도 비밀번호 인증 추가

## 2. 현재 사용자 여정

기본 흐름:
1. 언어 선택
2. 닉네임 입력 및 진입
3. 게임 카테고리 선택
4. 카테고리별 모델 목록 확인
5. 개별 게임 플레이
6. 평가 제출
7. 전체 비교 평가 페이지 확인
8. 코멘트/댓글 상호작용
9. 마이페이지에서 통계/배지 확인

보조 흐름:
- About 화면 이동
- 문의하기 링크 이동
- 개인정보처리방침 페이지 이동
- 관리자 인증 후 블라인드 처리

## 3. 화면 구성

### 3-1. 온보딩 영역

화면:
- 언어 선택 화면
- 닉네임 입력 화면
- 게임 카테고리 선택 화면

특징:
- 세 화면은 슬라이드 방식으로 전환
- 언어 -> 닉네임 -> 카테고리 방향은 위로 올라가는 느낌
- 뒤로 가기는 반대 방향
- GSAP 사용, 미로딩 시 CSS fallback 사용

### 3-2. 게임 모델 선택 화면

주요 요소:
- 선택한 게임 카테고리 타이틀
- 게임 설명 문구
- 해당 카테고리의 모델 카드 목록
- 모델별 플레이 수
- 사용자가 이미 평가한 모델 여부 표시
- 전체 비교 평가 보기 버튼

동작:
- 특정 모델 카드 선택 시 게임 플레이 화면으로 이동
- 전체 비교 평가 보기 버튼 클릭 시 결과 화면 이동

### 3-3. 게임 플레이 화면

주요 요소:
- 뒤로 가기 버튼
- 게임 설명 문구
- 실제 게임 iframe 또는 HTML 렌더 영역
- 평가 항목 6개
- 코멘트 입력란
- 평가 제출 버튼
- 해당 모델 기존 코멘트 영역

현재 평가 항목 레이아웃:
- 3열 2행 고정
- 큰 화면에서도 1행으로 늘어나지 않도록 고정

현재 평가 항목 명칭:
- 창의성
- 완성도
- 조작감
- 몰입감
- 재미
- 프롬프트 반영도

특이사항:
- 괄호가 포함된 항목은 줄바꿈 처리
- 플레이 진입 시 조회 기록 저장
- ESC 키로 게임 종료 안내 문구 존재

### 3-4. 전체 비교 평가 화면

주요 요소:
- 결과 테이블
- 평균 점수 정렬 헤더
- 최신순 / 좋아요순 댓글 정렬
- 코멘트 카드 목록
- 댓글 펼치기/접기
- 댓글 등록
- 좋아요/싫어요

특징:
- 코멘트 관련 인터랙션은 전체 페이지 리렌더 대신 부분 갱신
- 스크롤 위치 유지
- 버튼 클릭 시 상단으로 튀지 않도록 보정
- 화면 깜빡임 완화

### 3-5. 마이페이지

주요 요소:
- 닉네임
- 현재 프로필 배지
- 게임 종류별 평가 제출 수
- 가장 많이 조회한 게임 종류
- 가장 많이 조회한 모델 Top 5
- 유니크 평가 모델 수
- 배지 컬렉션

배지 컬렉션 정책:
- 획득 배지 표시
- 미획득 배지는 `?` 카드 표시
- 획득한 배지 중 하나를 프로필 배지로 선택 가능

특징:
- 마이페이지 전용 색상 테마 사용
- 영어 설정 시 게임 카테고리명도 번역된 표시명 사용

### 3-6. About / 문의 / 정책 화면

About 화면:
- 서비스 소개
- 문의하기 버튼

문의:
- Google Forms 기반 외부 문의 경로 사용

개인정보처리방침:
- 별도 화면 존재
- 푸터 링크로 진입
- 한영 지원
- 현재 서비스 처리 정보 + 향후 AdSense 도입 가능성 고지

## 5. 닉네임 정책

현재 정책:
- 영문 6~14자
- 한글 3~8자
- 영문 전용 또는 완성형 한글 전용
- 혼합 입력 차단
- 금지어 차단
- 관리자 오인 표현 차단
- 초성/중성 비정상 입력 차단

현재 구조적 한계:
- 닉네임 소유권 검증 없음
- 동일 닉네임을 아는 누구나 해당 닉네임으로 진입 가능
- 현재는 표시용 식별자에 가까움

## 6. 평가 및 코멘트 정책

평가:
- 모델별 게임 플레이 후 제출
- 코멘트 최소 길이 검증
- 동일 모델 반복 코멘트 작성 제한
- 제출 쿨다운 존재

코멘트:
- 좋아요 / 싫어요 가능
- 닉네임당 코멘트당 1개 반응
- 재클릭 시 취소
- 반대 반응 시 전환

댓글:
- 댓글 등록 가능
- 최소 길이 검증
- 10초 쿨다운
- 20시간 내 최대 50개 제한
- 펼치기/접기 지원

블라인드:
- 관리자만 코멘트/댓글 블라인드 가능
- 일반 사용자에게는 블라인드 문구만 표시

## 7. 배지 시스템

배지 구조:
- 획득형 + 선택형
- 서버에서 조건 계산
- 프로필 배지는 DB 저장

현재 배지 조건:
- 평론가 부엉이: 3종류 게임의 모든 모델 평가 완료
- 박사 부엉이: 4종류 게임의 모든 모델 평가 완료
- 교수 부엉이: 5종류 게임의 모든 모델 평가 완료
- GOAT 부엉이: 전체 모델 평가 완료
- 게이머 부엉이: 전체 조회수 100회 이상
- 마법사 부엉이: 던전 탐색 12개 평가 완료
- 군인 부엉이: 1인칭 미니 FPS 12개 평가 완료
- 마리오 부엉이: 횡스크롤 액션 12개 평가 완료
- 카드마스터 부엉이: 카드배틀 12개 평가 완료
- 우주선 부엉이: 서바이벌 디펜스 12개 평가 완료
- 보드 부엉이: 강화된 벽돌깨기 12개 평가 완료

UI 정책:
- 배지 획득 시 팝업
- 조회수 기반 배지도 즉시 팝업
- 배지명 강조
- PNG/SVG 배지 자산 혼용
