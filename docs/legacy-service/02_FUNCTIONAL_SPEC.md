# Functional Specification

## 1. 온보딩

### 1.1 언어 선택
- 사용자는 한국어/영어 중 하나를 선택할 수 있다.
- 선택한 언어는 `localStorage`에 저장된다.
- 언어 변경 시 게임 목록도 해당 언어 기준으로 다시 로드된다.

### 1.2 닉네임 진입
- 사용자는 닉네임을 입력해 서비스에 진입한다.
- 닉네임은 프런트와 서버에서 모두 검증된다.
- 현재 구조에서 동일 닉네임 소유권 검증은 없다.
- 특정 관리자 닉네임인 경우 추가 비밀번호 인증이 뜰 수 있다.

### 1.3 카테고리 선택
- 사용자는 게임 카테고리를 선택한다.
- 카테고리는 `strict` 그룹과 `advanced` 그룹으로 나뉘어 표시된다.
- 이후 모델 목록 화면으로 이동한다.

## 2. 게임 목록

- 선택한 카테고리의 모델 카드 목록을 보여준다.
- 각 모델 카드에는 플레이 수와 평가 여부 등이 반영된다.
- 카테고리 설명 문구를 노출한다.
- 전체 비교 평가 화면으로 이동할 수 있다.

## 3. 게임 플레이

- 선택한 블라인드 모델 게임을 렌더한다.
- 플레이 진입 시 조회 기록을 저장한다.
- 전체화면 전환 버튼을 제공한다.
- 평가 항목 6개와 코멘트 입력란을 제공한다.
- 해당 모델의 기존 코멘트 목록을 함께 보여준다.

## 4. 평가 제출

- 사용자는 점수와 코멘트를 제출할 수 있다.
- 최소 길이, 반복 작성 제한, 쿨다운 검증이 있다.
- 제출 성공 시 팝업을 보여준다.
- 제출 후 사용자 평가 이력과 마이페이지 데이터가 갱신된다.

## 5. 전체 비교 결과

- 선택 카테고리의 전체 결과를 테이블로 보여준다.
- 평균 점수 정렬이 가능하다.
- 댓글 영역에서 최신순/좋아요순 정렬이 가능하다.
- 코멘트 좋아요/싫어요, 댓글 펼치기/접기, 댓글 등록을 지원한다.
- 상호작용 시 화면 상단 점프 및 과도한 깜빡임을 방지한다.

## 6. 모델별 플레이 화면 코멘트

- 해당 모델의 기존 코멘트 목록이 보인다.
- 좋아요/싫어요를 누를 수 있다.
- 댓글을 펼치고 작성할 수 있다.
- 상호작용 시 전체 카드가 과도하게 깜빡이지 않도록 부분 갱신한다.

## 7. 반응 정책

- 닉네임당 코멘트당 하나의 반응만 허용한다.
- 같은 반응 재클릭 시 취소한다.
- 반대 반응 클릭 시 전환한다.

## 8. 마이페이지

- 현재 닉네임 기준 누적 통계를 보여준다.
- 게임 카테고리별 평가 수를 보여준다.
- 가장 많이 조회한 게임 종류를 보여준다.
- 가장 많이 조회한 모델 Top 5를 보여준다.
- 누적 유니크 평가 모델 수를 보여준다.
- 배지 컬렉션과 프로필 배지 선택 UI를 보여준다.

## 9. 배지

- 서버에서 배지 획득 조건을 계산한다.
- 획득한 배지는 컬렉션에 노출된다.
- 미획득 배지는 `?` 카드로 노출된다.
- 획득한 배지 중 하나를 프로필 배지로 설정할 수 있다.
- 새 배지 획득 시 팝업을 노출한다.

## 10. 관리자 기능

- 관리자 닉네임 + 비밀번호 인증 시 관리자 모드가 활성화된다.
- 결과 화면에서 코멘트/댓글 블라인드 토글이 가능하다.
- 일반 사용자에게는 블라인드 처리 문구만 보인다.

## 11. About / 문의 / 정책

- About 화면을 제공한다.
- 문의는 Google Forms 기반 iframe 내장 화면으로 처리하며, 현재 닉네임이 미리 채워질 수 있다.
- 개인정보처리방침을 별도 화면으로 제공한다.
- 정책 페이지는 한영 지원한다.

## 12. 현행 한계

- 닉네임 기반 구조는 인증 기능이 아니다.
- 계정/게스트 구분이 없다.
- 광고는 아직 실제 구현되지 않았다.
