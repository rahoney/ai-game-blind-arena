# Information Architecture

## 1. 상위 구조

```text
AI Game Blind Arena
├─ 온보딩
│  ├─ 언어 선택
│  ├─ 닉네임 입력
│  └─ 게임 카테고리 선택
├─ 콘텐츠 영역
│  ├─ 게임 모델 목록
│  ├─ 게임 플레이
│  ├─ 전체 비교 평가
│  ├─ 마이페이지
│  ├─ About
│  └─ 개인정보처리방침
└─ 공통
   ├─ 사이드바
   ├─ 푸터
   ├─ 팝업 모달
   └─ 관리자 상태
```

## 2. 화면 이동 구조

```text
언어 선택
  -> 닉네임 입력
    -> 카테고리 선택
      -> 게임 모델 목록
        -> 게임 플레이
        -> 전체 비교 평가
        -> 마이페이지
        -> About
        -> 개인정보처리방침
```

## 3. 내비게이션 구조

온보딩:
- 슬라이드 전환

콘텐츠:
- 사이드바 기반 이동
- 뒤로 가기 버튼
- 푸터의 개인정보처리방침 링크

## 4. 정보 계층

### 4.1 게임 관련
- 카테고리
- 블라인드 모델
- 실제 모델명
- 플레이 수
- 평가 여부
- 게임 HTML 자산

### 4.2 평가 관련
- 점수 항목 6개
- 코멘트 텍스트
- 제출 시간
- 카테고리
- 블라인드 모델 ID

### 4.3 상호작용 관련
- 좋아요/싫어요
- 댓글
- 블라인드 상태

### 4.4 마이페이지 관련
- 닉네임
- 현재 배지
- 획득 배지 목록
- 미획득 배지 placeholder
- 게임별 통계
- 모델별 통계

## 5. 신규 시스템에서 유지할 IA 원칙

- 온보딩은 단계형 구조 유지
- 게임 비교 흐름은 단절 없이 이어져야 함
- 결과/마이페이지/정책은 콘텐츠 레이어 안에서 독립 화면으로 유지
- 마이페이지와 결과 화면은 서비스 핵심 허브로 유지
