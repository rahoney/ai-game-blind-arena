# Screen Specification

## 1. 언어 선택 화면

목적:
- 서비스 진입 전 언어를 결정한다.

구성:
- 서비스 타이틀
- 언어 선택 버튼

주요 동작:
- 선택 언어 저장
- 다음 단계로 슬라이드 이동

## 2. 닉네임 입력 화면

목적:
- 사용자 식별용 닉네임으로 서비스에 진입한다.

구성:
- 닉네임 입력 필드
- 진입 버튼
- 언어 선택으로 돌아가기 버튼

예외:
- 닉네임 규칙 위반 시 메시지
- 관리자 닉네임일 경우 비밀번호 팝업

## 3. 카테고리 선택 화면

목적:
- 게임 종류를 선택한다.

구성:
- 카테고리 카드 목록
- `strict` / `advanced` 그룹 구분 제목과 설명
- 뒤로 가기 버튼

## 4. 게임 모델 목록 화면

목적:
- 카테고리별 모델 목록을 비교하고 진입한다.

구성:
- 카테고리 타이틀
- 게임 설명 문구
- 모델 카드 목록
- 전체 비교 평가 보기 버튼
- 사이드바

## 5. 게임 플레이 화면

목적:
- 실제 게임을 플레이하고 평가를 남긴다.

구성:
- 뒤로 가기 버튼
- 전체화면 버튼
- 게임 설명 문구
- 게임 렌더 영역
- 평가 항목 6개
- 코멘트 입력
- 평가 제출 버튼
- 기존 코멘트 영역

UI 규칙:
- 평가 항목은 3열 2행
- 코멘트 섹션 폭과 입력 영역 폭 정렬

## 6. 전체 비교 평가 화면

목적:
- 카테고리별 종합 비교 결과와 코멘트를 확인한다.

구성:
- 결과 테이블
- 정렬 가능한 점수 헤더
- 최신순 / 좋아요순 토글
- 코멘트 목록
- 댓글 영역

UI 규칙:
- 상호작용 시 스크롤 유지
- 깜빡임 최소화

## 7. 마이페이지

목적:
- 내 활동과 배지를 확인한다.

구성:
- 닉네임
- 현재 프로필 배지
- 게임별 통계
- 가장 많이 조회한 게임 종류
- Top 5 모델
- 유니크 평가 모델 수
- 배지 컬렉션

UI 규칙:
- 마이페이지 전용 색상 사용
- 미획득 배지 `?` 카드 표시

## 8. About 화면

목적:
- 서비스 소개와 문의 경로 제공

구성:
- 소개 문구
- 문의하기 링크

문의 화면 규칙:
- About 내부 액션으로 진입
- Google Forms가 iframe으로 내장된다.
- 현재 닉네임이 폼에 미리 전달될 수 있다.

## 9. 개인정보처리방침 화면

목적:
- 데이터 처리 및 향후 광고 관련 고지 제공

구성:
- 시행일
- 섹션형 정책 본문
- 뒤로 가기 버튼

진입:
- 푸터 링크

## 10. 팝업

종류:
- 평가 제출 성공
- 배지 획득
- 오류/안내
- 관리자 비밀번호 입력

규칙:
- 기본 alert 미사용
- 중앙 정렬
- 배지 획득 문구 강조
