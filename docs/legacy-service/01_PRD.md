# PRD

## 1. 제품명

- AI Game Blind Arena

## 2. 제품 목적

- 동일한 프롬프트로 생성된 여러 AI 모델의 게임 결과물을 블라인드 상태로 비교 평가한다.
- 사용자가 직접 플레이하고 정량 점수와 정성 코멘트를 남길 수 있게 한다.
- 누적 평가와 조회 이력을 바탕으로 마이페이지 통계와 배지 시스템을 제공한다.

## 3. 핵심 문제 정의

- AI 모델별 산출물 비교는 결과물 소비만으로는 공정성이 떨어진다.
- 모델 이름이 먼저 노출되면 편향이 개입될 수 있다.
- 단순 전시형 비교보다 실제 플레이 기반 비교가 더 설득력 있다.

## 4. 목표 사용자

- AI 모델 결과물 비교에 관심 있는 사용자
- 게임 생성 결과 품질을 체험적으로 판단하려는 사용자
- 특정 모델의 강점과 약점을 블라인드 환경에서 확인하려는 사용자

## 5. 핵심 가치 제안

- 블라인드 비교를 통한 편향 완화
- 실제 플레이 기반 평가
- 카테고리별/모델별 비교 경험 제공
- 누적 활동 기반 배지 및 통계 제공
- 커뮤니티형 코멘트/댓글 상호작용 제공

## 6. 현재 제품 범위

포함:
- 언어 선택
- 닉네임 기반 진입
- 게임 카테고리 선택
- 모델 목록 확인
- 게임 플레이
- 평가 제출
- 전체 비교 결과 확인
- 코멘트/댓글 반응
- 마이페이지 통계
- 배지 획득/선택
- 개인정보처리방침
- About / 문의하기
- 관리자 블라인드

제외:
- 정식 회원가입
- 비밀번호 기반 사용자 인증
- 이메일 인증
- 아이디/비밀번호 찾기
- 둘러보기 전용 권한 분리
- 광고 실제 게재

## 7. 성공 기준

정성 기준:
- 사용자가 카테고리별로 여러 모델을 직접 비교할 수 있어야 한다.
- 모델명을 모른 채 공정하게 평가하는 경험이 유지되어야 한다.
- 평가 후 결과 화면과 마이페이지에서 자신의 누적 활동이 명확히 보인다.

정량 후보 지표:
- 닉네임 생성/진입 완료율
- 카테고리 선택 후 플레이 진입율
- 플레이 후 평가 제출율
- 코멘트 작성율
- 마이페이지 방문율
- 배지 획득 사용자 수

## 8. 현행 제약

- 닉네임 기반 구조이며 사용자 소유권 검증이 없다.
- 프런트는 바닐라 JS 기반 정적 구조다.
- 계정/게스트 권한 모델이 없다.
- 광고는 아직 붙어 있지 않다.

## 9. 차기 제품 방향

- 신규 `AI_Game` 루트에서 프런트를 React + TypeScript + Vite로 재구축
- 닉네임 기반 구조를 정식 계정 + 둘러보기 구조로 전환 검토
- 도메인 확보 후 운영용 메일 및 정책 보강 검토
- AdSense 승인 이후 광고 위치 및 정책 최종 반영
